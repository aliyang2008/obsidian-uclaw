## 1. 产品概述

UClaw AI Chat 是一个 Obsidian 桌面插件，将 UClawDesktop 的 AI 对话能力嵌入 Obsidian 右侧面板。它通过本地 OpenClaw HTTP 网关与 AI 服务通信，支持多会话、文件附件（图片/文档）、流式输出打字机效果、以及 Markdown 渲染。

**核心特性：**

- 在 Obsidian 右侧边栏内直接进行 AI 对话
- 多会话（Tab）管理，消息本地持久化
- 支持上传图片、PDF、Markdown、Word 等文件
- 打字机效果模拟流式响应
- 与 Obsidian 主题 CSS 变量保持一致，自动适配深色/浅色模式
- 纯客户端实现，使用 Obsidian 内置 `requestUrl` 绕过 CORS/CSP 限制

## 2. 安装指南

### 2.1 手动安装（推荐）

1. 在 Obsidian 仓库中找到插件目录：
   ```
   {你的仓库}/.obsidian/plugins/
   ```
   如果 `plugins` 目录不存在，手动创建。

2. 将 `obsidian-uclaw` 文件夹完整复制到该目录下，确保以下文件结构：

   ```
   .obsidian/plugins/obsidian-uclaw/
   ├── main.js          ← 插件主体（编译产物）
   ├── manifest.json    ← 插件元数据
   └── styles.css       ← 样式文件
   ```

3. 在 Obsidian 中打开 **设置 → 第三方插件**。

4. 在"已安装插件"列表中找到 **UClaw AI Chat**，点击开关启用。

### 2.2 构建安装（开发者）

如果从源码构建：

```bash
cd obsidian-uclaw
npm install
npm run build
```

构建产物为 `main.js`，之后按照 2.1 步骤将三个文件放入插件目录即可。

## 3. 前置条件

UClaw AI Chat 依赖本地运行的 **OpenClaw 网关服务**。在启动插件前：

1. 确保 OpenClaw 网关已在本地运行（默认监听 `127.0.0.1:18790`）。
2. 网关需暴露以下接口：
   - `GET  /health` — 健康检查
   - `POST /v1/chat/completions` — 聊天补全（OpenAI 兼容 API）
3. 认证使用 Bearer Token，默认值为 `uclaw`。

> **提示**：如果网关端口不是默认的 18790，可在插件设置中修改（见 [第 6 节](#6-设置说明)）。

## 4. 界面概览

插件加载后会在 Obsidian 右侧边栏显示聊天面板，自上而下分为四个区域：

```
┌─────────────────────────┐
│  ● 已连接  [1] [2] [+][-] │  ← 头部：状态指示器 + 会话Tab
├─────────────────────────┤
│  AI: 你好，有什么可以…    │
│                         │  ← 聊天消息区（气泡样式）
│     你: 帮我总结这篇笔记   │
│                         │
├─────────────────────────┤
│  📎 photo.png  ×        │  ← 附件预览栏（上传后显示）
├─────────────────────────┤
│ [📎] [______________] [发送] │  ← 输入区
└─────────────────────────┘
```

| 区域 | 说明 |
|------|------|
| **状态指示器** | 显示网关连接状态：`● 已连接` / `○ 未连接` / `◌ 检测中…`。点击可手动重试 |
| **会话 Tab** | 数字编号的 Tab 页，支持多会话切换、新建（+）、删除（−） |
| **消息气泡** | 用户消息右对齐（紫色背景），AI 消息左对齐（灰色背景），系统消息居中（红色提示） |
| **附件栏** | 显示已选中的附件缩略标签，可逐个移除 |
| **输入区** | 文本框 + 📎附件按钮 + 发送/停止按钮 |

## 5. 基本操作

### 5.1 打开对话面板

有三种方式打开 UClaw AI Chat 面板：

| 方式 | 操作 |
|------|------|
| **Ribbon 图标** | 点击 Obsidian 左侧 Ribbon 栏的 💬 图标 |
| **命令面板** | `Ctrl/Cmd + P` → 输入"UClaw" → 选择 **打开 UClaw AI 对话** |
| **自动打开** | 插件启动时会在 Obsidian 布局就绪后自动激活面板 |

**关闭面板**：右键点击面板标题栏的视图图标 → 关闭；或直接关闭右侧边栏。

### 5.2 发送消息

1. 在底部文本框中输入内容。
2. 按 **Enter** 发送（`Shift + Enter` 换行）。
3. 等待 AI 以打字机效果逐字输出回复。
4. 发送过程中，按钮变为红色 **停止**，点击可中断当前请求。

**消息渲染支持：**

- **加粗**：`**文本**`
- *斜体*：`*文本*`
- `行内代码`：`` `代码` ``
- 代码块：` ```language\n...\n``` `

### 5.3 多会话管理

插件支持创建多个独立会话，每个会话有各自的消息历史和上下文。

| 操作 | 方法 |
|------|------|
| **新建会话** | 点击 Tab 栏的 **+** 按钮 |
| **切换会话** | 点击对应数字 Tab |
| **删除当前会话** | 点击 **−** 按钮（仅当会话数 ≥ 2 时显示） |

**会话持久化规则：**

- 会话列表存储在 `localStorage` 的 `uclaw-sessions` 键中。
- 每个会话的消息单独存储，键名为 `uclaw-msgs-{sessionId}`。
- 切换会话时自动保存当前会话消息再加载目标会话消息。
- 关闭视图时自动保存当前会话消息。
- 删除会话会同时清除该会话的消息存储。

**注意**：当前版本每个会话使用相同的智能体 ID（默认 `main`）。

### 5.4 附件上传

支持在消息中附加文件，通过 📎 按钮操作：

1. 点击输入区左侧的 **📎** 按钮。
2. 在文件选择器中选择文件。
3. 选中的文件会显示在附件预览栏中。
4. 发送消息时附件与文本一起提交。

**支持的文件类型：**

| 类别 | 扩展名 | 处理方式 |
|------|--------|----------|
| 图片 | `image/*` | 以 `image_url`（base64 dataUrl）格式发送给 AI |
| PDF | `.pdf` | 以文本描述形式发送（`[用户上传了文件: xxx.pdf]`） |
| 文档/数据 | `.txt`, `.md`, `.doc`, `.docx`, `.csv`, `.json`, `.xml` | 同上，文本描述形式 |

**附件操作：**

- 点击附件标签上的 **×** 可移除单个附件。
- 发送后附件列表自动清空。

**本地存储注意**：附件 dataUrl 不会写入 localStorage（避免撑爆存储），持久化时仅保存纯文本显示内容。

### 5.5 网关状态检测

状态指示器反映网关连接状况，并支持手动检测：

| 状态 | 显示 | 含义 |
|------|------|------|
| 已连接 | `● 已连接`（绿色） | 网关运行正常 |
| 未连接 | `○ 未连接`（灰色） | 网关无法访问 |
| 检测中 | `◌ 检测中…`（橙色脉冲动画） | 正在尝试连接 |

**手动检测：**

- 点击状态指示器文字。
- 或使用命令面板：`Ctrl/Cmd + P` → **检测 OpenClaw 网关连接**。

插件启动和视图打开时也会自动执行一次健康检查。

## 6. 设置说明

进入 **设置 → 第三方插件 → UClaw AI Chat → 齿轮图标** 打开设置面板。

| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| **网关端口** | `18790` | OpenClaw 本地网关的 HTTP 端口号（1–65535） |

修改端口后会自动重建 GatewayClient 实例。如需生效，关闭并重新打开聊天视图即可。

## 8. 技术架构

### 整体架构

```
┌───────────────┐     requestUrl      ┌──────────────────┐
│  Obsidian     │ ◄──────────────────► │  OpenClaw 网关    │
│              │   (Bearer: uclaw)     │                  │
│  ┌─────────┐ │                      │  GET  /health     │
│  │ main.ts │ │                      │  POST /v1/chat/   │
│  │ (Plugin)│ │                      │       completions │
│  └────┬────┘ │                      └──────────────────┘
│       │      │
│  ┌────▼──────────┐
│  │ UClawChatView  │ ← ItemView，负责 UI 渲染 + 消息持久化
│  └────┬──────────┘
│       │
│  ┌────▼──────────┐
│  │ GatewayClient  │ ← 封装 HTTP 通信，管理连接状态
│  └───────────────┘
└───────────────────────────────┘
```

### 通信协议

| 项目 | 详情 |
|------|------|
| 协议 | HTTP（通过 Obsidian 内置 `requestUrl`） |
| 端点 | `http://127.0.0.1:{port}/v1/chat/completions` |
| 方法 | POST |
| 认证 | `Authorization: Bearer uclaw` |
| 请求体 | `{ "model": "openclaw:main", "messages": [...], "stream": false }` |
| 响应格式 | OpenAI Chat Completions 兼容：`choices[0].message.content` |

### 流式模拟

网关请求使用 `stream: false`（非流式）模式，获取完整响应后在客户端以打字机效果模拟流式输出：每次输出 1–3 个字符，间隔 10–30ms 随机延迟。

### 持久化方案

所有数据通过浏览器 `localStorage` 持久化：

| 存储键 | 内容 |
|--------|------|
| `uclaw-sessions` | JSON 数组：`[{ id, name, agentId, createdAt }]` |
| `uclaw-msgs-{sessionId}` | JSON 数组：`[{ role, content, id }]` |

## 9. 故障排除

### 9.1 状态显示"未连接"

**可能原因：**

1. OpenClaw 网关未启动。
2. 网关端口与插件设置不一致。
3. 防火墙或安全软件阻止了本地 127.0.0.1 连接。

**解决步骤：**

1. 确认网关服务已运行：在浏览器中访问 `http://127.0.0.1:18790/health`。
2. 检查插件设置中的端口号是否与网关监听端口一致。
3. 点击状态指示器手动重试检测。
4. 查看 Obsidian 开发者控制台（`Ctrl+Shift+I`）是否有网络错误。

### 9.2 发送消息无响应

1. 确认状态为"● 已连接"。
2. 检查网关日志是否有请求记录。
3. 如果消息中包含大型附件（如图片 base64），可能导致请求超时或 payload 过大。
4. 尝试在新会话中发送简单文本测试。

### 9.3 插件未出现在已安装列表

1. 确认 `main.js`、`manifest.json`、`styles.css` 三个文件都在 `.obsidian/plugins/obsidian-uclaw/` 目录下。
2. 确认 `main.js` 是 `cjs` 格式的编译产物（不是 `.ts` 源文件）。
3. 重启 Obsidian 并检查"第三方插件"页面是否识别到插件。

### 9.4 消息历史丢失

消息存储在浏览器的 `localStorage` 中。如果清除了 Obsidian 的缓存数据或使用了隐私模式，历史消息会被清除。

## 10. 开发与构建

### 环境要求

- Node.js ≥ 18
- npm ≥ 9

### 项目结构

```
obsidian-uclaw/
├── main.ts              ← 插件入口（Plugin 生命周期、设置、命令注册）
├── UClawChatView.ts     ← 聊天视图（ItemView、UI 构建、消息管理、附件处理）
├── GatewayClient.ts     ← 网关客户端（HTTP 通信、状态管理、回调）
├── styles.css           ← 样式（适配 Obsidian CSS 变量）
├── manifest.json        ← 插件元数据
├── package.json         ← npm 配置
├── tsconfig.json        ← TypeScript 配置
└── esbuild.config.mjs   ← esbuild 构建脚本
```

### 开发命令

```bash
# 安装依赖
npm install

# 开发模式（监听文件变化自动编译 + sourcemap）
npm run dev

# 生产构建（压缩 + 类型检查）
npm run build
```

### 构建细节

- 使用 **esbuild** 打包，输出 CommonJS 格式（`main.js`）。
- `obsidian` 和 `electron` 标记为 external，由 Obsidian 运行时提供。
- 目标平台：`es2020`。
- 生产构建开启 `minify`，不生成 sourcemap。
